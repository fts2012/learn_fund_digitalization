## 第6课：投资运作流程（前台）

---

### 对话讲解

#### 一、先立骨架：投研决策链

基金公司前台的运转逻辑，一条链四个节点：

```
研究员 ──→ 基金经理 ──→ 交易员 ──→ 风控合规
  │            │            │           │
 研究支持    组合决策     指令执行    事前/事中拦截
```

| 角色 | 干什么 | 和你IT相关的系统 |
|------|--------|----------------|
| **研究员** | 覆盖行业/公司，出研究报告、盈利预测、目标价 | 研报系统、数据终端（Wind/Choice/Bloomberg） |
| **基金经理** | 资产配置、组合构建、择时择券 | OMS（订单管理系统） |
| **交易员** | 指令执行、最优成交 | O32/恒生投资交易系统 |
| **风控合规** | 比例检查、禁投池、关联交易 | 事前风控引擎（通常嵌在O32里） |

关键认知：**研究员不交易、交易员不决策。** 基金经理下指令，交易员只负责执行。这条隔离是防止利益输送的第一道防火墙。

#### 二、O32在基金公司——和你在券商看到的不一样

你熟悉恒生生态，O32在券商自营/资管里是核心交易系统。在基金公司，它有额外三层约束：

**券商O32用法**：
- 自营交易员直接操作，风控参数自己设
- 事后稽核为主，事前检查相对灵活（自营盘子小、品种多）

**基金O32用法**：
- 基金经理只能「下指令」，不能「执行」——有权限隔离
- 指令发出前，必须过**事前合规风控引擎**，不是选项、是硬拦截
- 风控参数由**独立的风控部门**设置，投资部门无权修改

一句话：**券商O32是交易工具，基金O32是合规执行系统。**

事前合规风控引擎检查什么？每笔指令至少过：

| 检查项 | 规则来源 | 
|--------|---------|
| 持股比例（双十规定） | 《运作管理办法》第32条 |
| 禁投池 | 公司内部名单（关联方、ST、评级过低个股） |
| 反向交易 | 同公司不同基金之间不能互为对手方 |
| 公平交易 | 同一天对同一标的，不同基金的买卖方向不能相反 |
| 集中度 | 单日交易量不超过该股票日均成交量的X% |

**这个引擎为什么是「不可逾越的红线」？**

不是技术原因，是制度原因。券商可以事后说"这个交易事后看看有没有问题"——公募不行。原因很简单：

- 公募的信息披露要上季报，季报里持仓清清楚楚。如果某只基金超了10%上限，全市场都看得见
- 托管行每天复核。你踩线了，托管行的报告直接到证监会，不经过你
- 持有人诉讼风险。如果因为比例超标导致亏损，证监会的处罚函就是持有人发起民事诉讼的关键证据

所以基金公司的O32风控参数是**硬编码级**的严肃性——不是提醒弹窗，是直接拒绝指令下发。

#### 三、指令流程：从基金经理脑到交易所，一分钟内发生了什么

```
9:35 基金经理决定买入100万股茅台
   ↓
9:35 在OMS录入指令：代码600519，数量1,000,000，限价或市价
   ↓
9:35 事前风控引擎自动跑：
      × 茅台占该基金净值？→ 8.2%，未超10% ✓
      × 茅台在禁投池？→ 否 ✓
      × 同公司其他基金有卖出茅台？→ 否 ✓
      × 该基金经理今日已有反向操作？→ 否 ✓
   ↓
9:36 风控通过，指令到达交易员终端
   ↓
9:36 交易员选择执行策略（TWAP/VWAP/冰山/直接下单）
   ↓
9:36 指令经O32 → 交易所报盘机 → 上交所
   ↓
9:37 成交回报 → O32 → 估值系统（次日生成持仓）
```

整个过程，从基金经理头脑到交易所，不到两分钟。但中间有三个系统节点，每个节点都可能出问题：

1. **风控引擎参数配置错误**：应该拦截的没拦截
2. **O32与估值系统数据不同步**：风控用O32的实时持仓算比例，但O32的持仓数据可能和估值系统有差异（T+1到账的债券、分红除权等）
3. **交易员策略选择失误**：TWAP在流动性差时可能打不出去，市价单在极端行情可能滑点

#### 四、事前 vs 事后：基金和券商风控哲学的根本差异

| 维度 | 券商（事后监控） | 公募（事前拦截） |
|------|----------------|-----------------|
| **风控时点** | T+1检查（交易已经发生） | T时刻实时阻断（交易未发生） |
| **违规后果** | 问责、罚款、系统整改 | **根本不会发生违规** |
| **业务影响** | 交易体验不被中断 | 可能错失交易机会（指令被拒） |
| **系统复杂度** | 事后跑批，性能要求相对低 | 毫秒级实时计算，不能拖慢交易 |
| **误拦容错** | N/A（事后不存在误拦） | 误拦=基金错过交易窗口=持有人损失 |

为什么基金必须事前拦截？回到底层逻辑——**信义义务**。你代为管理的不是自己的钱，你没有任何资格"先交易、后检查"。这也是第2课讲的：管理人对持有人的义务是「正向标准」，不是「负面清单」。

#### 五、投资决策的「三道防线」

这是面试高频题。基金公司投资流程有三道防线：

```
第一道防线：投资部门自身（基金经理自己有合规意识，不踩红线）
第二道防线：风控/合规部门独立监控（事前风控引擎的硬拦截）
第三道防线：内部审计/稽核（事后定期检查流程是否合规）
```

对应IT系统：

- 第一道：OMS里的辅助提示（"该股占净值已达9.8%，接近10%上限"）——不是拦截，是提醒
- 第二道：O32风控引擎硬拦截——拒绝执行
- 第三道：内审系统的稽核日志——所有被拦截的指令都有记录

### 课后作业

1. **指令追踪**：找一家基金公司的季报（任何股票型基金均可），看它的前十大持仓。假设你是这只基金的风控人员，列出每个重仓股的「占净值比」，判断是否触发双十规定。

2. **风控场景**：某股票基金规模10亿，今日已买入茅台3000万（占净值3%），基金经理又要加仓8000万茅台。请问：
   - O32风控引擎会对第二笔指令做什么？
   - 如果基金经理坚持要买，他有什么合法途径？还是没有？

3. **预习**：简单了解TA系统（中国结算的登记过户系统）、估值核算系统的基本概念。下节课第7课讲运营链条（后台），这是第二阶段的业务地基。
