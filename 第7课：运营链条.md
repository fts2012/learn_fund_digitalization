## 第7课：运营链条（后台）

---

### 对话讲解

这是第二阶段「数据层」的地基课。学完第7课，你就能理解持仓数据、净值数据、绩效数据为什么长成那样。

---

#### 一、后台运营四大系统，一张图说清关系

```
投资者买入基金，钱和信息的流向：

  投资者
    │ 申购款
    ↓
  销售机构（银行/蚂蚁/天天基金）
    │ 汇总申购数据 + 资金
    ↓
  TA系统（登记过户）         ← 基金公司或中国结算
    │ 确认份额、登记持有人
    ├──→ 估值核算系统        ← 基金公司
    │      │ 计算净值
    │      ↓
    │   托管行复核            ← 托管银行
    │      │ 复核通过
    │      ↓
    │   信息披露              ← T+1公布净值
    │
    └──→ 资金清算系统        ← 基金公司/托管行
           │ 交收资金、支付费用
           ↓
        托管行完成资金划转
```

---

#### 二、TA系统（登记过户）——基金的"账户管家"

TA = Transfer Agent，登记过户系统。是基金后台的第一个入口。

**TA做什么**：

| 功能 | 具体内容 |
|------|---------|
| **份额登记** | 记录每个持有人在每只基金的份额数量 |
| **申赎处理** | T日15:00前申请的申购/赎回，生成确认数据 |
| **分红处理** | 现金分红或红利再投资的份额变动 |
| **账户管理** | 持有人开户、销户、信息变更 |
| **费用计算** | 申购费、赎回费的计算和分配 |

**自建TA vs 中登TA**：

| | 自建TA（基金公司自己建） | 中登TA（中国结算） |
|------|--------------------------|---------------------|
| **适用产品** | 场外开放式基金 | ETF、LOF等场内产品 |
| **复杂度** | 较高（要自己维护） | 相对标准 |
| **控制力** | 完全自主 | 受中登规则约束 |
| **市场占比** | 大多数基金公司 | 主要在交易所产品 |

**TA系统的一天**：

```
15:00  截止接收当日申赎申请
15:00-16:00  汇总销售机构的申请数据
16:00-17:00  计算确认份额、费用
17:00-18:00  生成确认数据，发给估值系统（用于净值计算）
18:00-19:00  生成对账文件，发给托管行
T+1日上午  发送确认数据给销售机构（投资者看到份额确认）
```

---

#### 三、估值核算系统——净值的"出生地"

估值核算是基金后台最核心的系统。它每天回答一个问题：**这只基金今天值多少钱？**

**估值系统的输入**：

| 数据 | 来源 | 时点 |
|------|------|------|
| 持仓数据 | O32交易系统 | 收盘后T+0 |
| 收盘价 | 交易所/估值机构 | 收盘后 |
| 申赎数据 | TA系统 | T日下午 |
| 费用计提 | 内部参数 | 每日固定 |

**估值流程**：

```
① 获取T日持仓：从O32导入当日交易后的最新持仓
② 获取行情数据：收盘价、第三方估值（债券用中债/中证估值）
③ 计算资产总值：∑(持仓数量 × 价格) + 现金 + 应收款
④ 计算负债：应付管理费 + 应付托管费 + 应付交易费用 + 其他
⑤ 计算净值：资产总值 - 负债 = 基金资产净值
⑥ 计算份额净值：净值 ÷ 总份额 = 单位净值
⑦ 生成估值表：发送给托管行复核
⑧ 托管行复核确认（或不确认）：双方一致 → 净值生效
```

**债券估值的特殊性**：

股票用收盘价就完了。债券不是——大量债券成交不活跃，没有实时公允价格。所以债券必须用**第三方估值**：

- **中债估值**（中债登发布）：覆盖银行间债券市场
- **中证估值**（中证指数公司发布）：覆盖交易所债券

第三方估值不是可选项，是**证监会强制要求**——债券基金必须使用第三方估值，不能自己拍价格。

这对IT意味着：估值系统必须有每日自动接入第三方估值数据的接口。中债/中证的数据格式、下发时间、异常处理都是硬需求。

#### 四、清算交收——钱怎么到账

基金后台的交易清算链：

```
T日：基金经理下单买入股票
  ↓
T日：O32记录成交（成交价格、数量、金额）
  ↓
T日收盘后：O32导出成交数据 → 估值系统（记账），中央结算公司（证券交收）
  ↓
T+1：资金交收（托管行从基金账户划钱到卖方）
  ↓
T+1：证券到账（卖方证券划到基金的证券账户）
```

三个关键时间点，你IT的人要记住：

| 类型 | 资金交收 | 证券交收 |
|------|---------|---------|
| A股股票 | T+1 | T+1 |
| 债券（银行间） | T+0或T+1 | T+0或T+1 |
| QDII海外 | T+2或更长 | T+2或更长 |

---

#### 五、信息披露——净值怎么从估值系统到你手机上

估值系统算出净值 → 托管行复核 → 双方确认 → 发布。时间线：

```
估值核算完成 ──→ 托管行复核 ──→ 双方一致 ──→ 信息披露
   T日18:00        T日19:00        T日20:00     T+1日上午
```

披露渠道：基金公司官网、证监会指定的信息披露媒体、销售机构同步。

---

#### 六、托管行复核——不是走形式，真的有来回

托管行怎么复核净值？**不是对照管理人给的数字点"确认"。** 托管行有自己的估值系统，独立算一遍。双方结果一致 → 确认；不一致 → 查差异、定位问题、修正后重新比对。

日常差异来源：

- 停牌股估值方法不同（管理人用指数收益法，托管行用最近收盘价）
- 债券估值数据版本不同
- 费用计提口径差异
- 申赎份额数据延迟

每条差异都可能触发估值调整和净值追溯修正——这是后续第11课持仓数据和第12课净值数据的核心内容。
